# Context Handoff

Use this skill to make work portable without dragging entire transcripts into
future sessions.

## Workflow

1. Determine mode: save, restore, update, or compact.
2. For save/update, use [references/handoff-template.md](references/handoff-template.md).
3. For restore, use [references/resume-checklist.md](references/resume-checklist.md).
4. Store handoffs under `knowledge/handoffs/` unless the user gives another
   location.
5. Keep handoffs factual: goals, decisions, files, commands, evidence, open
   questions, and next actions.

## Rules

- Do not include secrets, credentials, private customer data, or unnecessary raw
  logs.
- Prefer links to files and commands over pasted transcript.
- Mark uncertain claims explicitly.
- Include exact dates for time-sensitive status.
- Keep handoffs short enough to read at session start.
