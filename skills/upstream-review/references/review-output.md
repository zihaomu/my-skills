# Review Output

Use this format for upstream PR review.

```markdown
## Blockers

- [severity] file:line - Issue.
  Evidence:
  Risk:
  Suggested fix:
  Required test:

## Maintainer Readiness

- Scope:
- Tests:
- Docs:
- Benchmarks:
- Backend compatibility:

## Suggested PR Text

<short maintainer-facing summary>

## Verification Matrix

| Area | Command | Result | Gap |
| --- | --- | --- | --- |
```

Keep tone direct and useful. Avoid stylistic nits unless they affect review
success or maintainability.
